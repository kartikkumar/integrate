5 
6 integrate is a header-only, C++ (template) library of numerical integrators to solve Ordinary Differential Equations (ODE). A CMake module is available to make it easy to include integrate in other projects: [Findintegrate.cmake](https://github.com/openastro/cmake-modules/blob/master/Modules/Findintegrate.cmake).

15 Requirements
16 ------
17 
18 To install this project, please ensure that you have installed the following (install guides are provided on the respective websites):
19 
20  - [Git](http://git-scm.com)
21  - A C++ compiler, e.g., [GCC](https://gcc.gnu.org/), [clang](http://clang.llvm.org/), [MinGW](http://www.mingw.org/)
22  - [CMake](http://www.cmake.org)
23  - [Doxygen](http://www.doxygen.org "Doxygen homepage") (optional)
24  - [Gcov](https://gcc.gnu.org/onlinedocs/gcc/Gcov.html) (optional)
25  - [LCOV](http://ltp.sourceforge.net/coverage/lcov.php) (optional)
26 
27 integrate does not depend on any libraries. The following libraries are optional (see `Build options`):
28 
29  - [Catch2](https://github.com/catchorg/Catch2) (unit testing library necessary for `BUILD_TESTS` build option)
30 
31 These dependencies will be downloaded and configured automagically if not already present locally (requires an internet connection).

33 Installation
34 ------
35 
36 Run the following commands to download, build, and install this project.
37 
38  git clone https://www.github.com/openastro/integrate
39  cd integrate
40  git submodule init && git submodule update
41  mkdir build && cd build
42  cmake .. && cmake --build .
43 
44 To install the header files, run the following from within the `build` directory:
45 
46  make install
47 
48 Note that dependencies are installed by fetching them online, in case they cannot be detected on your local system. If the build process fails, check the error log given. Typically, building fails due to timeout. Simply run the `cmake --build .` command once more.

50 Build options
51 -------------
52 
53 You can pass the following, general command-line options when running CMake:
54 
55  - `-DCMAKE_INSTALL_PREFIX[=$install_dir]`: set path prefix for install script (`make install`); if not set, defaults to usual locations
56  - `-DBUILD_DOXYGEN_DOCS[=ON|OFF (default)]`: build the [Doxygen](http://www.doxygen.org "Doxygen homepage") documentation ([LaTeX](http://www.latex-project.org/) must be installed with `amsmath` package)
57  - `-DBUILD_TESTS[=ON|OFF (default)]`: build tests (execute tests from build-directory using `ctest -V`)
58  - `-DBUILD_DEPENDENCIES[=ON|OFF (default)]`: force local build of dependencies, instead of first searching system-wide using `find_package()`
59 
60 The following commands are conditional and can only be set if `BUILD_TESTS = ON`:
61 
62  - `-DBUILD_COVERAGE_ANALYSIS[=ON|OFF (default)]`: build code coverage using [Gcov](https://gcc.gnu.org/onlinedocs/gcc/Gcov.html) and [LCOV](http://ltp.sourceforge.net/coverage/lcov.php) (both must be installed; requires [GCC](https://gcc.gnu.org/) compiler; execute coverage analysis from build-directory using `make coverage`)
63 
64 Pass these options either directly to the `cmake ..` command, e.g., to build the tests:
65 
66  cmake -DBUILD_TESTS=on ..
67 
68 N.B.: Toggling options to build tests using `ccmake` does not work correctly, as the necessarily libraries are not download automagically!

70 Project structure
71 -------------
72 
73 This project has been set up with a specific file/folder structure in mind. The following describes some important features of this setup:
74 
75  - `cmake/Modules` : Contains `CMake` modules, including `Findintegrate.cmake` module
76  - `docs`: Contains code documentation generated by [Doxygen](http://www.doxygen.org "Doxygen homepage")
77  - `include/integrate`: Project header files (*.hpp)
78  - `scripts`: Shell scripts used in [Travis CI](https://travis-ci.org/ "Travis CI homepage") build
79  - `test`: Project test source files (*.cpp) that are provided to the [Catch2](https://github.com/catchorg/Catch2 "Catch2 Github repository") framework
80  - `.travis.yml`: Configuration file for [Travis CI](https://travis-ci.org/ "Travis CI homepage") build, including static analysis using [Coverity Scan](https://scan.coverity.com/ "Coverity Scan homepage") and code coverage using [Coveralls](https://coveralls.io "Coveralls.io homepage")
81  - `CMakeLists.txt`: main `CMakelists.txt` file for project (should not need to be modified for basic build)
82  - `Dependencies.cmake`: list of dependencies and automated build, triggered if dependency cannot be found locally
83  - `Doxyfile.in`: [Doxygen](http://www.doxygen.org "Doxygen homepage") configuration file, adapted for generic use within project build (should not need to be modified)
84  - `LICENSE.md`: license file for project
85  - `ProjectFiles.cmake`: list of project source files to build
86  - `README.md`: project readme file, parsed as main page for [Doxygen](http://www.doxygen.org "Doxygen homepage") documentation
